# Лабораторная работа 1 - Класс Product

**Вариант:** 3 (Интернет-магазин)

---

## Вопрос №1. Что является сущностью?

Сущностью в данной работе является **Товар (Product)** в рамках предметной области «Интернет-магазин». Класс `Product` моделирует реальный товар, который можно продавать, отслеживать его остатки, применять скидки и управлять его доступностью.

---

## Вопрос №2. Какие атрибуты будут у данного объекта?

В классе реализованы два типа атрибутов.

**Атрибут класса:**
- `currency` (валюта) — строка со значением по умолчанию `"RUB"`. Является общим для всех товаров магазина и доступен как через сам класс `Product.currency`, так и через любой экземпляр.

**Атрибуты экземпляра:**
- `_name` — название товара
- `_price` — базовая цена
- `_remains` — остаток на складе
- `_discount` — текущая скидка (в процентах)
- `_status` — статус товара
- `_product_id` — уникальный артикул
- `_active` — флаг активности (в продаже или снят)

Для защиты данных реальные значения хранятся в защищённых полях (с нижним подчёркиванием), а доступ к ним снаружи предоставляется через публичные свойства (`@property`).

---

## Вопрос №3. Какие инварианты?

В классе реализованы следующие инварианты, которые проверяются через модуль `validate.py`:

| Инвариант | Правило |
|-----------|---------|
| Цена (`price`) | Число (`int` или `float`), строго больше 0 и не более 99 999 |
| Остаток (`remains`) | Целое число, больше или равно 0 |
| Скидка (`discount`) | Целое число от 0 до 99 включительно |
| Название (`name`) | Непустая строка |
| Активность (`active`) | Если товар неактивен, бизнес-методы блокируются |

Объект не позволяет записать в себя некорректные данные — при нарушении любого инварианта выбрасывается ошибка `ValueError`.

## Вопрос №5. Есть ли состояния?

Да, у объекта есть два состояния: активный и неактивный.

- **Активный** (`_active = True`) — товар в продаже. Его можно купить и узнать цену.
- **Неактивный** (`_active = False`) — товар снят с продажи. Купить его нельзя.

**Как управлять состоянием:**

```python
